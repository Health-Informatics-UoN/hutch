# Contributing

Thanks for your interest in contributing! We welcome contributions of all kinds, including:

- Bug reports and new issues
- Code improvements and new features
- Documentation updates and fixes

## Submitting Contributions

To contribute:

- Check open issues, and create or comment on an issue to volunteer to work on it.
- A maintainer will then assign the issue to you if it is available and appropriate to work on.
- Fork the repository and create a branch for your changes.
- Follow existing coding patterns and keep contributions focussed.
- Open a pull request (PR) with a clear description of your changes.

## Copyright & Licensing

By contributing, you agree that your contributions are licensed under the same MIT License as the project. This means your work can be freely used, modified, and distributed under the same terms.

## Code of Conduct

We don’t have a formal code of conduct yet, but the rule is simple: Be nice.

Be respectful, constructive, and assume good intent.

## Need Help?

If you're unsure about anything, open an issue or ask in a discussion. We’re happy to help!
